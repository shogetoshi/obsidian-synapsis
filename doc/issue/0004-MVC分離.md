🔴 最重要（High Priority）

1. View層の分離：HTMLをPythonから分離

- 問題: index() に330行以上のHTML/CSS/JSが埋め込まれている（228-560行目）
- 影響: フロントエンドのテスト・保守が困難、バージョン管理も複雑
- 対策:
  - templates/ ディレクトリを作成
  - Jinja2テンプレートエンジンを導入
  - 静的ファイル（CSS/JS）を static/ に分離

2. Service層の導入：ビジネスロジックをコントローラから分離

- 問題: save_file() と ask_ai() がファイルI/O、AI呼び出し、Git操作を直接実行
- 影響: ユニットテスト不可、責務が不明確
- 対策:
services/
├── ai_service.py      # AI処理
├── file_service.py    # ファイル操作
└── git_service.py     # Git操作

3. ask_ai() エンドポイントの分割（特に重要）

- 問題: 1つの関数で5つの責務を持つ（バリデーション、プロンプト構築、AI呼び出し、ファイル書込み、Git操作）
- 影響: 変更時の影響範囲が大きすぎる
- 対策: 各責務を独立したサービスに分割
